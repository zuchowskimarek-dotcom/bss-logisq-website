# 1. Homepage (`/`)

## 1.1 Content

### What works

- Clear central claim: **"Warehouse Execution for Shorter Project Runtimes"**
- Problem framing ("late failure due to reinterpretation") is strong and differentiated
- Digital Twin correctly demoted (no longer core cause)

### Issues

- Homepage still explains *too much* for a SELL layer
- Some sections drift into EXPLAIN territory (product structure hints, technology terms)

### Change suggestions

- Remove or shorten any content that answers *"how"* instead of *"why"*
- Keep homepage focused on:
    - project risk
    - time-to-market
    - execution predictability

Add **one short clarifying sentence only** (max 1 line):

> "LogisQ-WES coordinates warehouse execution across people, automation, and systems in real time."
> 

This anchors newcomers without diluting the funnel.

### What works

- Strong differentiation vs. generic WMS marketing
- No "AI / optimization / magic" claims

### Issues

- "Built for fully automated warehouses" (or similar wording) narrows perception too early

### Change suggestions

- Replace exclusivity with inclusivity:
    - from "built for fully automated warehouses"
    - to "designed for manual, automated, and mixed environments"

---

## 1.3 Visual

### What works

- Clean layout, good whitespace
- CTA placement is logical

### Issues

- Hero visuals don't yet *explain execution*
- Visual story still feels "platform-ish"

- Avoid diagrams that look like architecture here
